Сортировка выбором
Сама судь
Сортировка выбором похоже с сортировкой вставками, которая разделяет массив на сортированный и несортированный подмассивы. Но здесь отличается тем, что будущие сортированные числа, на котором стоят указатели (что за указатели объясню позже), меняются местами с минимальными числами справа.

Итак, давайте возьмём, как обычно, несколько элементов массива:

0 8 3 1 2

Давайте ещё обозначем числа в виде стили шрифта:
m - указатель числа, который будет заменять с минимальным числом справа.
n - минимальное число.

Числа, которые находятся до указателя -- отсортированные (левые части). Сам указатель не является отсортированным, поскольку ещё неизвестно, будет ли меняться число, или нет. Числа, которые находятся после указателя, неотсортированные (правые части).

Итак, давайте разберёмся, как работает алгоритм. Мы берём самое 1-ое число и ставим его как указатель. Далее мы находим минимальное число, которое является неотсортированным. Теперь мы видим такую картину:

0 8 3 1 2

Как ни странно, число 0 и является минимальным неотсортированного элемента. Но, поскольку здесь стоит указатель, то здесь нету смысла менять местами. Тут же само на себя не поменяешь местами. После того, как отсортируются элементы, мы передвигаем указатель на одно число вперёд. И тут же берём минимальное число неотсортированных элементов. И следующая картина будет такова:

0 8 3 1 2

Вот теперь здесь можно менять местами, и результат будет следующий:

0 1 3 8 2

И так мы делаем до конца. Следующие действия будут таковы:

0 1 3 8 2
0 1 2 8 3
0 1 2 8 3
0 1 2 3 8
0 1 2 3 8
0 1 2 3 8

Если указатель находится на самом последнем элементе, то наш алгоритм завершён. Это и есть сортировка выбором.
Объяснение кода
Итак, наша сортировка будет начинаться с цикла с счётчиком (for). В значения for'а ставим всё по стандарту.

Внутри for мы введём несколько переменных: в переменную min мы сразу ставим ЧИСЛО, в котором стоит указатель (что за указатель, я пояснил выше. Смотрите саму судь), с помощью которого мы будем находить минимальное число неотсортированных элементов массива; в переменную minID мы указываем НОМЕР элемента, числа которого будет минимальным. Эта переменная нужна для того, чтобы заменить число в дальнейшем.

После того, как мы ввели переменные мы ставим ещё цикл с счётчиком (for). В значения for'а тоже ставим по стандарту, но только начальное значение счётчика будет j = i + 1. То есть j -- это номер элемента, которое находится на 1 элемент правее i. Если поставить j = i, то ошибка (как мне кажется) не будет, но нежелательно так ставить, ибо нету смысла проверять элемент массива в дальнейшем. 2-ой for нужен для того, чтобы найти минимальное число неотсортированных элементов массива.

Внутри for мы ставим проверку, если число из правой части (неотсортированных элементов) будет меньше переменной min, то в переменных ставим число (min) и номер (minID) элемента массива. В противном случае ничего выполняться не будет.

Объясню по подробнее, чтобы было понятно.

Теперь мы выходим из 2-го цикла for и сортируем. Мы введём в временную переменую temp, значение которой будет равнятся число, на котором стоит указатель (это arr[i]):

Код: int temp = 8;
Пример: 0 8 3 1 2

Затем в указателе значение ставим минимальное число (min):

Код: arr[i] = min;
Пример: 0 1 3 1 2

А вместо минимального числа (arr[minID]) приравниваем число из временной переменной (temp):

Переменная: temp = 8
Код: arr[minID] = temp;
Пример: 0 1 3 8 2

И так каждый раз, 1-ый цикл будет работать, пока указатель не достигнет до самого конца.
